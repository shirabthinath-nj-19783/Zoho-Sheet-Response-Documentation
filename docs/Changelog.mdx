<h2>Cell Style Definition</h2>

- In earlier versions, the Number Format Response (Pattern Response) was located at the 9th index of each CELL_DATA array under CELL_META -> META -> COLUMN_DATA. It has now been moved to the 15th index in the Cell Style Definition.
- To avoid sending the same pattern response for each cell, it has been moved to the cellStyleDefinition.

<h2>Cell Meta</h2>
- Previously, the Number Format Response (Pattern Response) was sent at the 9th index within CELL_DATA. It has now been moved to the CELL_STYLE_DEFINITION response.
- As a result, all subsequent indexes in the CELL_DATA response have been shifted forward by one position.
- The displayValue will no longer be sent for Picklist, Boolean, and Error values since their actual value and displayValue are always identical.
- For Float, Percentage, Currency, Fraction, and Scientific Formats the formulaString will be sent only if the decimalSeparator is not a dot (.). This is because, when the decimalSeparator is a dot, the actual value and formulaString are identical.
- For Date and DateTime formats, the formulaString will be sent only if the DateFormat is not MDY. This is because the actual value is already in the MDY format.
- For String types, the formulaString will be sent only if the value is prefixed with an apostrophe ('). This is because only in such cases do the actual value and formulaString differ.
- The picklist response at the 17th index should be used instead of the one at the 11th index.

<h2>Filters</h2>
- The filter response key <b>FILTER_DETAILS</b> has been changed to <b>NEW_FILTER_DETAILS</b>.
- Previously, the filteredRows response was sent within the Filter response. It has now been moved to the Sheet Meta.
- Additionally, we have added the ID, FILTER_VIEW_ID, and DISPLAY_FILTER_BUTTONS keys to the filter response.
- In the FilterData response, we have replaced the string-based keys with CommandConstant keys. Check [here](../docs/Structure/Sheet%20Meta/Filters.mdx) for the new filterData keys and structure. 
- Additionally, we have included Picklist details in the FilterData response to support the picklist filter.
