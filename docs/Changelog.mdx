<h2>Cell Style Definition</h2>

- In earlier versions, the Number Format Response (Pattern Response) was located at the **9th** index of each **CELL_DATA** array under **CELL_META -> META -> COLUMN_DATA**. It has now been relocated to the **15th** index within the cell style definition.
- To avoid duplicating the same pattern response for every cell, it has been centralized under cell style definition.

<h2>Cell Meta</h2>
- Previously, the Number Format Response (Pattern Response) was sent at the 9th index within CELL_DATA. It has now been moved to the CELL_STYLE_DEFINITION response.
- As a result, all subsequent indexes in the CELL_DATA response have been shifted forward by one position.
- The displayValue will no longer be sent for Picklist, Boolean, and Error types, as their actual value and display value are always identical.
- For Float, Percentage, Currency, Fraction, and Scientific formats, the formula string will be sent only if the decimal separator is not a dot (.), since the actual value and formula string are identical when the separator is a dot.
- For Date and DateTime formats, the formula string will be sent only if the dateFormat is not MDY, because the actual value is already in MDY format.
- For String types, the formula string will be sent only if the value is prefixed with an apostrophe ('), as this is the only case where the actual value and formulaString differ.
- The picklist response previously located at index 11 should now be retrieved from index 17.

<h2>Filters</h2>
- The filter response key **FILTER_DETAILS** has been renamed to **NEW_FILTER_DETAILS**.
- Previously, the **filteredRows** response was included within the filter response. It has now been moved to the Sheet Meta section.
- The filter response now also includes the following additional keys: **ID, FILTER_VIEW_ID, and DISPLAY_FILTER_BUTTONS**.
- In the FilterData response, string-based keys have been replaced with CommandConstant keys. Refer to the updated key structure [here](../docs/Structure/Sheet%20Meta/Filters.mdx).
- Picklist details have been added to the FilterData response to support picklist-based filtering.
